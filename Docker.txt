Docker Basic - 

# Container is like a virtual machine 

# Docker is advance version of virtualization, its make containerization

# containerization is a advance version of virtualization

# Docker is an open source contralise platform designed to create, deploy and run application

# Docker uses containers on the host OS to run application to use the same linux kernel as a system on the host computer, rather than creating a whole virtual OS

# we can install docker on any OS but docker engine run natively on linux distribution

# Docker written in "GO" language 

# Docker is tools that perform OS level virtualization, also know as containerization

-----------------------

Docker Advantanges - 

# No pre allocatio of ram 

# Docker enables you to build a containers images and use that same image across every step of the deployment process

# less cost , light weight, 

# it can run on physical machine, Hardware, or cloud 

# you can re-use the image 

# it took very less time to create container

-----------------------------------------

Disadvantage Docker -

# Docker is not a good solution for application that requires rich GUI

# Diffucult to manage large amount of container 

# Docker does not provide cross-platform compatibility means if an application is designed to run in a docker container on windows then it cant run an linux or vice versa 

# docker is suitable when the develiopment OS and testing OS are the same if the OS different we should use use VM

# No solution for data recovery and backup

Docker Eco system - set of software or packages

1. Docker Client - work palce 
2. Docker Daemon or server or docker engine - image to container convert
3. Docker Hub registry - image save place 
4. Docker Images - like AMI, template 
5. Docker Compose -

#  Docker Daemon (docker engine)- Docker daemon runs on the host os, it is responcibale for runing conatiner to manage docker service, and docker daemon can communicate with other daemon

# Docker Client -  Docker can interact with docker daemon through a client(CLI)
	- Docker client uses command CLI and Rest API to communicate with the docker daemon
	- when a client runs any server command on the docker client terminal, the client terminal these docker command to the docker daemon
	- it is possible for docker client to communicate with more than one daemon
	
Docker Host -  is used to provide an environment to excute and run application, it contains the docker daemon, images, containers, network and storage 
	
Docker Hub / Registery -  Docker registry manage and shares the docker images 
	- two type of registry
	-  public registery - is also called as docker hub
	-  private registery  it is used to share images within the enterprise

Docker Commands

# docker images =  to check stored images 

# docker search (image name)  - eg, docker search jenkins =  find out images in docker hub

# docker pull (image name)  - eg, docker pull jenkins = download image from dockerhub to local machine

# run is combination of create and start  also, -it means - interactive mode terminal, --name- used for provide a name for the container 

# docker run -it --name testdocker ubuntu /bin/bash =  pull the image ubuntu and run the container with name testdocker 

# docker ps =  check runing container

# docker ps -a =  check all containers (running and stop)

# docker ps -aq =  check all containers (running and stop) it will show only container id 

# docker start (container name) = to start container

# docker attach (container name)  =  login into the container (need container stared)

# docker exec -it (container name) /bin/bash = login into the container with new process (need container stared)

# docker stop (container name) =  stop the container (you should be outside of container and conatiner should be stop state) 

# docker rm (container name) = remove container   

# docker rm -f (container name) = forcefully remove conatiner

# docker rmi (imagename) = remove docker image 

---------------------------------------------------------------

# docker diff (containername) - check the chnages from previous image to current state

# docker commit (containername) (imagename) = create image from container

Dockerfile -  dockerfile is basically a test file it contains some set of instarctions, which helps automation of docker image creation 
	- D capital in docker file name

Dockerfile components - 

# FROM = for base image this command must be on top of the dockerfile

# RUN = to excute command it will create a layer in image

# MAINTAINER = auther/ owner/ description

# COPY = copy file from local system (docker vm) we need to provide source, destination we cant download file from internet and any remote repo

# ADD = similar to copy but its provide feature to download files from internet also we extract file at docker image side

# EXPOSE = to expose ports such as port 8080 for tomcat. port 80 for ngnix etc 

# WORKDIR = to set working directory for a container 

# CMD =  execute commands but during container creation 

# ENTRYPOINT = similar to CMD but has higher priority over CMD, first command will be executed by ENTRYPOINT only

# ENV = environment variable 

# ARG = 

e.g. 

FROM ubuntu
WORKDIR /tmp
RUN echo "This is vivek machine" > /tmp/vivek.txt 
COPY  a1.txt /tmp

Volume 

# volume is simple a direvtory inside our container

# firstly  we have to declare the directory as a volume and then share volume 

# even if we stop container, still we can access volume

# volume will be created in one container

# you can declare a directory as a volume  only while creating container

# you cant create volume from existing container 

# you can share one volume across any number of container

Benifits of volume - 

# Decoupling container from storage 

# share volume among different containers

# attached volume to containers 

# on deleting container volume does not delete 

----------------------------------------------------

$ Creating volume from docker file e.g.

FROM ubuntu
VOLUME ["/myvolume1"]		(docker file)

docker build -t myimage . 	( create image from dockerfile)

docker run -it --name newcontainer myimage /bin/bash ( create container from image)

Now share volume with another container 
docker run -it --name container2 --privileged=true --volumes-from newcontainer ubuntu /bin/bash 

Note - voulme share should create only at the time of container create
and or declare in dockerfile 

$ Createb volume by using command and share with another conatiner at time 2nd container create
docker run -it --name con3 -v /volume1 ubuntu /bin/bash 

docker run -it --name con4 --privileged=true --volumes-from con3 ubuntu /bin/bash
