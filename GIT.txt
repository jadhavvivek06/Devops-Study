# Git -  Source code management tool (Software tool)
		- git is primaraly used for code managment, which acts like a version control system and which allow to do collabration  

# Git is completely free 

# There is a 2 type - 
	- Centralised version control system (old)(CVCS)
	- Distributed Version control system 

# Drawbacks - CVCS
	1. It is not locally available, meaning you always need to be connected to a network to perform any action
	2. Since everything is centralised, if cerntral server gets failed, you will lose entire data

# Distributed Version control system -  Created by Linus Torvald - 2005
- In Distributed version control system every contributer has a local copy or clone of the main repository i.e everyone maintain a local reporsitory of their own which contain all the files and metadata present in main repository. 

# Git install - install git on machine

# git init -  git initilize -  it will create .git folder  and convert in reporsitory

		  2. Staging area

	- Working directory --> staging area			     =  add
	- Staging area --> local repo 						 =  commit

	- github / gitlab  ---> local repo					 =  Pull

cherry-pick - pick the perticular commit and merge into other branch
	- Cherry picking is the act of picking a commit from a branch and applying it to another. git cherry-pick can be useful for undoing changes. For example, say a commit is accidentally made to the wrong branch. You can switch to the correct branch and cherry-pick the commit to where it should belong.
